# Rule: Re-send After Decline Allowed

**Applies to:** `flows/send-connection-request.md`

## What the rule says

After a connection request is declined, the sender CAN send a **new**
connection request to the same profile. The previously-declined profile
appears in Add Connection search in an active/selectable state (not
greyed-out, no "Connected" or "Pending" label).

## Notes

- Verified for Funder Org, Implementer Org, and Both Org.
